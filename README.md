# SETTING Archive Custom Node
균일한 품질 유지를 위한 SETTING 아카이브
  
이 커스텀 노드는 사전 입력한 SETTING과 프로젝트명, 저장 위치를 시계열로 기록하여 단순한 개인의 컨텐츠 생성이 아닌 팀 단위의 작업에 용이하도록 제작된 노드입니다.

## 주요 특징
1. 기능 추가의 용이함
기본 프리셋: 기존 게임 개발에 사용하는 6개 에셋(배경, 구조물, 인물, 특수효과, 장비, 음원)과 그에 필요한 SETTING 값

커스터마이징: 사용자 고유의 제작 방식에 맞춰 기능 변경 가능 (DIR 동적 참조)

2. 간단한 노드 구조 (Master-Slave 구조)
- SETTING NODE: 컨텐츠 생성을 위한 파라미터 및 환경 설정

- MASTER NODE (저장): 설정된 값을 저장하고 시계열 관리함

- SLAVE NODE (분배): 저장된 설정 값을 수신하여 각각의 생성 워크플로우로 전달

3. 동적 UI 및 아카이브 저장 형태
- order_list.txt를 통해 UI 명칭과 순서를 제어

- DIR 구조와 config.json 생성을 통해 사용자가 원하는 설정 구현 가능

- SETTING 외 포함 항목: 프로젝트명, 시계열 타임스탬프, 저장 경로 등

### 미구현
현재 개발 중인 미구현 기능:

- SETTING NODE 종류의 동적 참조

- MASTER NODE의 INPUT 동적 참조

- SLAVE NODE의 RETURN 동적 참조

(현재는 에셋 명칭 수정을 위해 직접 복사/붙여넣기 작업이 필요)

#### 기술적 고민과 개선 방향
발견된 문제점
- 기존의 커스텀 노드들이 제작자, 목적성에 따라 다양한 형태를 가지는데 이를 하나로 묶어서 관리하려다 보니 기존 노드들과의 연결성에서 차이가 발생.

- 실제 생성 노드로 직접 연결하는데 부적합함을 확인하였습니다.

##### 향후 해결 방안
- 기능의 단순화: 순수 **프로젝트 관리 전용 노드**로 사용.
  
**OR**
  
- 확장: 기존 노드에 적합한 **전용 생성 노드** 추가 제작 검토.

###### 확인점
- 최근의 AI 기반 컨텐츠 생성 방식이 굉장히 고도화되었음을 확인
- 기존 커스텀 노드들이 개개인의 사용자가 원하는 방향으로 정확하게 일치시키기 어려워 억지로 비틀어서 써야하고 실질적으로 기능의 일부만 사용되는 노드들이 많아 워크플로우 화면이 복잡해지고 커스텀 노드 라이브러리 또한 지저분해짐을 확인
- 팀 단위의 협업 환경에서 사용을 한다면 어느정도 사용 툴의 범위를 지정하여 생성 노드를 직접 작성하는 방향이 장기적으로 옳음을 확인

####### 최종목표  
매번 코딩해야만 하는 동적 참조가 아닌 내부 기능 추가 관련 가이드(AI 추천)가 존재하는 형태로 제작하여 정말 올인원 노드가 되는 것이 목적chive Custom Node
