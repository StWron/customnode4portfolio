해당 커스텀 노드는 ai를 통한 컨텐츠 생성에 있어 균일한 품질을 유지하기 위하여 입력한 설정을 시계열에 따라 기록하기 위해 만들어진 커스텀 노드입니다.
default로 일반적인 게임 개발의 6개 에셋(배경,구조물,인물,특수효과,장비,음원)으로 작성을 전제로 작성되었으며 차후 추가될 동적 참조 기능으로 종류의 변경이 쉽습니다
크게 SETTING NODE와 MASTER(저장) SLAVE(분배)의 역할을 가지고 있으며 현재 SETTING NODE 내부의 동적 UI는 이미 구축되어 있습니다
동적 UI는 각 에셋별 폴더 안의 order_list.txt에 ui 명을 입력하고 동일한 명칭의 폴더를 만든 뒤 원하는 파일이나 설정(config.json)을 내부에 생성하여 구현할 수 있습니다
SETTING NODE 도 차후 동적으로 추가가 쉽게 새로 작성할 예정입니다
원래 목적은 컨텐츠 생성 세팅 값을 시계열과 프로젝트 명으로 저장하는 기능까지만 구현하고 나머지는 기존의 커스텀 노드를 사용할 예정이였으나 제가 원하는 형태로 구현하려면 생성 노드까지 제가 작성해야 하는 점 확인했습니다
