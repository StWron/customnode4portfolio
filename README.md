#AI 컨텐츠 생성 보조용 세팅 값 아카이브 노드
현재 보고 계시는 커스텀 노드는 ai를 통한 컨텐츠 생성에 있어 균일한 품질을 유지하기 위하여 입력한 설정을 시계열에 따라 기록하기 위해 만들어진 커스텀 노드입니다.
##특징
- 1.default로 일반적인 게임 개발의 6개 에셋(배경,구조물,인물,특수효과,장비,음원)으로 작성되게 제작이 되었으나 사용자가 사용하던 제작 방식으로 변경 가능합니다.(DIR 동적 참조)
- 2.SETTING NODE와 MASTER(저장) SLAVE(분배)의 역할을 가지고 있으며 SETTING NODE에서 세팅을 한뒤 MASTER에서 저장하고 SLAVE에서 수신하여 분배하는 방식입니다.
- 3.동적 UI는 order_list.txt에 명칭과 순서를 입력하고 동일한 명칭의 폴더를 만든 뒤 원하는 파일이나 설정(config.json)을 내부에 생성하여 구현할 수 있습니다
- 4.작성된 SETTING은 프로젝트명, 시계열, 저장 위치를 포함하여 저장됩니다

###미구현
현재 SETTING NODE 자체의 생성, 수정과 MASTER NODE의 INPUT, SLAVAE NODE의 RETURN 동적 참조 미구현 상태입니다. 현재는 직접 복사/붙여넣기하여 에셋 명칭을 수정하여야 합니다.

####문제점
- 기존 설계 방향은 SETTING만 저장하고 나머지는 기 존재하는 노드에 SETTING 값만 분배하는 방식으로 시계열, 프로젝트명, 저장 위치로 전체를 관리만 하는 형태로 진행하였으나 기존 커스텀 노드들의 사용 방향과 다른 점을 확인했습니다.
- 기존의 노드들은 목적, 제작자 등에 따라 굉장히 자유분방하게 흩어져 있는데 제 커스텀 노드의 설계 사상은 모든 걸 하나로 관리하는 쪽에 가까워서 발생하는 문제로 보입니다.

#####해결방법
-프로젝트 관리용 노드로 사용
-OR
-생성 노드 제작

######반성 및 느낀점
-현재의 고도화되고 복잡해진 AI 컨텐츠 제작을 너무 쉽게 보고 막연하게 접근하여 바로 생성용으로 연결하기 힘든 노드가 되었습니다.
-하지만 사용 및 학습하면서 기존 커스텀 노드들이 제작자 작업 스타일, 하드 코딩, 버전 업데이트의 불안정성 등의 부분을 볼때 작업자가 원하는 워크플로우의 원활한 구축을 위해선 직접 제작하는 편이 장기적으로 좋은 부분 확인하였습니다.
